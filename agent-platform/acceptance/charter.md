# 验收团队测试章程 — 榕器 · Agent 人机协作平台

## 目的

验证平台能否支撑《AI数智化企业应用推广行动方案》真正落地：**每个角色的人都能在没有培训的情况下完成自己的核心任务**（低学习门槛），数据口径与方案一致，关键业务链路（派活→交付→审核→度量→治理）闭环可用。

## 测试环境

- 服务：`http://127.0.0.1:8000`（常驻部署，uvicorn + SQLite）
- 界面走查：无头 Chrome 截图（`?person=<id>#/<视图>` 一键身份直达）
- 业务旅程：真实 API 调用（各人用自己的 token）
- 视图 hash：dashboard / workspaces / agents / scenarios / tasks / skills / knowledge / org / governance / roadmap

## 数据口径基准（来自行动方案，验收必查）

- 首年现金投入 **34.3 万** = 算力 13.79 万（阿里云 2.385 + 智谱 8.4 + 灵活采购 3）+ NAS 10.5 万（采购 9.5 + 运维 1）+ 创新激励 10 万
- 直接收益 **54 万/年**；综合收益 79 万；首年净 ROI 57.5%
- 五类数字员工方向；首批 5 个试点（外贸跟单/会议纪要/BOM/质量异常/研发测试）
- 组织：5 平台（43/34/177/64/17 人）28 部门；教练团 4 人、开发者候选 30 人

## 评分与判定

- 学习门槛评分 1–5（5 = 完全不用教就能用）；同时回答"我是否愿意每天用"
- 问题分级：**阻塞**（核心链路走不通）/ **严重**（数据错误或严重误导）/ **一般**（体验困惑、文案问题）/ **建议**（可更好）
- 角色判定：能落地 / 修复后可落地 / 不能落地
- **合格线**：无阻塞、无严重；董事长与徐露璐评分 ≥4；其余 ≥3

## 迭代流程

第 N 轮：8 角色并行体验 → 报告存 `round<N>/<角色>.md` → 汇总去重排优先级 → 修复（coder）→ 回归复测（重点角色 + 上一轮问题项）→ 直至全部达到合格线。
