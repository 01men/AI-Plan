### ② 任务中心"只看待我审核"开关 — 部分修复 ⚠️

- 第一轮问题 6（建议）：无"我的待审"视角。
- 本轮实测：任务中心左上新增"只看待我审核"复选框（截图 `daishuan_tasks.png` 可见），前端过滤逻辑为 `status==='待审核' && reviewer_id===myId`（/static/app.js:967），空列提示文案为"没有待你审核的任务，去协作空间 @数字员工 派一个吧"——**开关本身存在、交互与文案都做得清楚**。
- 但**实际不好用，根源在后端派单逻辑没改**：
  - API 实测 `GET /api/tasks?reviewer_id=6` → **0 条**；全部任务（含其他验收人新建）均 `reviewer_id == creator_id`（实测 7/7）。
  - `engine._pick_reviewer`（app/engine.py:217-225）第一个分支即 `if creator_id: return creator_id`——审核人恒等于任务创建人，任何任务都不会派给"别人"审核。
  - 由此产生逻辑死结：开关过滤出的"待我审核"任务，只能是我自己创建的任务；而审核接口 `POST /api/tasks/{id}/review` 明确禁止审核自己发起的任务（app/routers/tasks.py:110-111，403"不能审核自己发起的任务"）。**即这个开关能筛出来的，恰恰是我无权审的；我能审的，永远筛不进来。**
- 结论：UI 层面的开关做了（点赞空态文案），但只要派单逻辑仍是"审核人=创建人"，该开关对任何用户都无实际用途。判定**部分修复 ⚠️**，建议后端 `_pick_reviewer` 改为按部门/教练团指派非创建人的审核人。未评级为阻塞：不勾选时全局看板正常可用。
- 另观察到：看板各列计数、卡片信息（优先级/数字员工/创建人/截止/点击审核→）正常；并发建单下渲染无异常。
