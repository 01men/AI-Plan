### ④ 技术硬伤评估

- ❌ **并发（阻塞级）**：顺序请求完全正常（GET /api/agents 平均 7ms），但一旦并发就成片 500——
  - 5 路并发 GET `/api/people`：5/5 全部 500 Internal Server Error
  - 3 路并发 GET：3/3 正常；2 路并发写 PATCH：4/4 全部 500
  - 10 线程 20 次并发写：20/20 全部 500，0.19s 内集体失败
  - 字面的 `database is locked` 出现 **0 次**（服务端把它包成了无信息的 500），但根因显然是 SQLite 单连接/锁竞争。65 人同时用的场景下这是硬伤：早会一起打开页面就会有人看到报错。
- ⚠️ **空状态（一般）**：API 层正常（不存在的空间返回 `[]`、200）；但 UI 层没有空态设计——数字员工卡片直接显示"累计任务 0 / 节省工时 0h"，治理列表若空就是一张裸表头，没有"还没有数据，先去 xxx"的引导。
- ⚠️ **加载反馈（未能完整验证）**：静态截图看不到骨架屏/loading 态，headless 环境下页面 12s 预算内渲染完整；真实弱网下的加载表现本次未覆盖，存疑。
- ⚠️ **移动端 800×600（一般）**：左侧导航不折叠，固定吃掉约 29% 宽度；空间卡片自适应为 2 列尚可读，但底部"空间文档"五列表格在窄屏必然横向挤压，无横向滚动提示也无卡片化降级（证据：fandingxin_mobile.png）。

## 问题清单

1. [阻塞] 全局/后端 — 期望：多人并发访问正常；实际：≥5 路并发读、≥2 路并发写即全部 500（无 "database is locked" 明文，只有 Internal Server Error）。65 人规模不可用，上线前必须修（SQLite WAL + 连接池，或换库）。
2. [严重] API `PATCH /api/agents/{id}` — 期望：status 只接受枚举值，非法值 4xx；实际：`"飞天状态"` 返回 200 直接入库，列表页会显示裸字符串徽章（测试脏数据已清理）。
3. [一般] 治理中心 — 期望：红线、审计日志作为治理核心应一眼可见；实际：藏在第 3/4 页签，首屏是激励列表。
4. [一般] API 422 报文 — 期望：面向非技术用户的中文错误；实际：pydantic 英文原文外抛（login 参数错误、query 类型错误）。
5. [一般] 权限 — 期望：developer 只能改本部门数字员工；实际：可改任意部门（含董事办），仅有登录鉴权没有资源级授权。
6. [一般] 移动端窄屏 — 期望：侧栏可折叠/表格降级；实际：800px 下侧栏占 29%，表格无滚动提示。
7. [建议] 知识库 — L4 密级无样例文档，四级体系演示不完整。
8. [建议] 空状态 — 列表/卡片无空态引导文案，0 值直接裸露。
9. [建议] API 路径命名 — `/api/governance/audits` 与直觉命名（audit-logs/audit）不一致，无重定向。
