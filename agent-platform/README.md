# 榕器 · Agent 人机协作平台

金华聚杰电器《AI数智化企业应用推广行动方案》的落地支撑底座：一个 AI 原生的人机协作平台，把 65 个数字员工当作"正式员工"纳入 5 平台 / 28 部门的组织架构，覆盖从场景立项、派活执行、人工审核到 KPI 度量、费用治理的完整闭环。

**AI 原生人机协作理念**

- **沟通即执行**：在协作空间的 Agent 执行区发言或 `@数字员工`，即自动派发任务并产出交付物，无需切换系统。
- **Agent 是员工**：数字员工有组织归属、有责任人、有技能清单、有 KPI（任务量/节省工时/准确率），与人类员工同台管理。
- **三区交互**：每个工作区分为讨论区（公开研讨）、Agent 执行区（派活与交付审核）、私聊打磨区（一对一打磨需求与提示词）。
- **人在环路审核**：数字员工的交付物一律"待人工审核"，通过才计入绩效，驳回则自动重做；写回动作必须人工确认并留痕。
- **心跳主动服务**：项目管理智能体每 6 小时自动巡检一次（也可手动触发），在经营驾驶舱发布运营日报并催办 24 小时内临期的任务。

## 一、一键启动

**前提**：Windows + Python 3.10 及以上（已在 Python 3.13 验证），并安装依赖：

```bash
pip install fastapi uvicorn
```

**方式一（推荐）**：双击项目根目录下的 `启动平台.bat`，服务启动后约 3 秒自动打开浏览器。

**方式二（命令行）**：

```bash
cd agent-platform
python -m uvicorn app.main:app --port 8000
```

然后访问 <http://localhost:8000>。

首次启动会自动创建 `data/platform.db` 并播种演示数据（48 名员工、65 个数字员工、81 个场景、20 个 Skill、6 个知识空间等），无需任何手工初始化。

## 二、演示身份与权限

登录页从组织人员中选择身份，免密登录（演示环境），token 存于浏览器 localStorage。

| 层级 | 人数 | 代表身份 | 权限差异 |
| ---- | ---- | -------- | -------- |
| `boss` 董事长 | 1 | 董事长 | 全部功能 + 任务审核 + 报销审批 |
| `coach` 教练团 | 4 | 李乐平 / 师圆圆 / 付玉虎 / 莫柳金 | 全部功能 + 任务审核 + 报销审批 |
| `backbone` 业务骨干 | 4 | 各平台业务负责人 | 全部功能 + 任务审核 + 报销审批 |
| `developer` 开发者 | 30 | 胡鑫（国际销售部）等 AI 应用开发者 | 可派活、申报场景/激励/报销、查看全部看板，**看不到**审核与审批按钮 |
| `staff` 使用人 | 9 | 一线使用人员 | 同开发者，仅日常使用 |

> 审核与审批按钮（任务通过/驳回、报销三级审批）仅 `boss` / `coach` / `backbone` 三层可见，体现"人在环路"的权责设计。建议演示路径：先用开发者"胡鑫"派活，再切换董事长或教练审核。

## 三、十大功能导览

1. **驾驶舱**：KPI 总览（场景覆盖率、审核通过率、节省工时、年化效益）、14 天趋势、四波次进度、数字员工排行榜、实时动态流，可一键手动触发心跳。
2. **协作空间**：三区交互的 IM 式工作区，@数字员工即派活，交付物卡片内直接审核。
3. **数字员工**：65 个数字员工台账，按平台/波次/状态/类别筛选，查看画像、绑定场景、近 14 天指标与近期任务。
4. **场景库**：81 个应用场景清单（含预期效益与动作拆解），支持敏捷申报与一键立项——立项即自动创建项目工作区并拉入关联数字员工。
5. **任务中心**：待处理/进行中/待审核/已通过/已驳回五列看板，人工审核通过后自动计入数字员工绩效，驳回后引擎自动重做一轮。
6. **Skill 库**：可复用技能资产库（公开/组织/个人三级），被数字员工引用的技能计入复用 KPI。
7. **知识库**：6 套群晖 DS925+ NAS 空间，文档按 L1-L4 密级分级管理。
8. **组织通讯录**：5 平台 28 部门 48 人的组织树，人与其名下数字员工同屏呈现。
9. **治理中心**：激励申报（火花奖/银齿轮奖/金扳手奖/种子基金）、大模型 token 费用报销三级审批（平台长→数字化复核→财务）、全量审计留痕、六大红线公示。
10. **路线图**：筑基/推广/深化三阶段 15 个里程碑与四波次（8-11 月）推进计划。

## 四、接入真实大模型

平台默认使用**模板模拟**生成交付物（不联网、零成本、可离线演示）。执行引擎在每次生成交付物时读取 `settings` 表，若以下三个键齐全，即自动切换为真实大模型调用（OpenAI 兼容接口）：

| 键 | 说明 | 示例（智谱 GLM） |
| -- | ---- | ---------------- |
| `llm_base_url` | OpenAI 兼容接口地址（不含 /chat/completions） | `https://open.bigmodel.cn/api/paas/v4` |
| `llm_api_key` | 接口密钥 | `你的 API Key` |
| `llm_model` | 模型名 | `glm-4-flash` |

配置方法（任选其一）：

```bash
sqlite3 data/platform.db
```

```sql
INSERT OR REPLACE INTO settings(key,value) VALUES('llm_base_url','https://open.bigmodel.cn/api/paas/v4');
INSERT OR REPLACE INTO settings(key,value) VALUES('llm_api_key','你的 API Key');
INSERT OR REPLACE INTO settings(key,value) VALUES('llm_model','glm-4-flash');
```

- 任何 OpenAI 兼容服务均可（智谱 GLM、DeepSeek、通义千问、Kimi 等），只改三个值即可。
- **无需重启**：引擎每次生成时实时读取配置；调用超时或失败会静默回落到模板模拟，不会报错中断。
- 删除这三个键（`DELETE FROM settings WHERE key LIKE 'llm_%';`）即恢复纯模板模拟。

## 六、数据重置

删除 `data/platform.db`，重新启动服务（或再次双击 `启动平台.bat`）即可自动重新建表并播种，恢复到初始演示状态。种子数据规模：5 平台 / 28 部门 / 48 人 / 65 数字员工 / 81 场景 / 20 Skill / 6 知识空间 / 15 里程碑。

## 七、技术栈

- **后端**：Python + FastAPI + SQLite（标准库 `sqlite3`，无 ORM、无外部服务依赖）；心跳为 `asyncio` 后台任务，每 6 小时执行一次。
- **前端**：原生 HTML/JS/CSS 单页应用（无构建步骤），Tailwind CSS 与 ECharts 走 CDN；登录 token 存 localStorage。
- **大模型适配**：标准库 `urllib` 调 OpenAI 兼容接口，未配置时纯模板模拟，可完全离线运行。
- **接口约定**：见 `API.md`（全部端点、请求体与真实响应示例）。
